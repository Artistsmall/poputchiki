# Попутчики (простое веб‑приложение)

Прототип приложения **«Попутчики»** с двумя ролями:

- **Водитель**: создаёт поездки (откуда, куда, во сколько), видит заявки пассажиров и может принимать/отклонять их. При принятии заявки в маршрут добавляется дополнительная точка (откуда/куда везти пассажира).
- **Пассажир**: указывает направление (откуда/куда хочет уехать), видит подходящих водителей и может отправить заявку выбранному водителю.

## Стек

- Backend: **Node.js + Express + SQLite** (персистентное хранение пользователей, поездок и заявок).
- Авторизация: **JWT** (регистрация/вход, роли водитель/пассажир, хеширование пароля через bcrypt).
- Frontend: **чистый HTML + CSS + JS**, отдаётся тем же Express‑сервером.

## Структура проекта

- `server/`
  - `index.js` — код сервера и API.
  - `package.json` — зависимости backend.
- `public/`
  - `index.html` — интерфейс приложения.
  - `app.js` — логика работы на клиенте.
  - `styles.css` — стили.

## API (кратко)

- **Авторизация**
  - `POST /api/auth/register` — регистрация (`name`, `email`, `password`, `role: driver|passenger`), возвращает `{ token, user }`.
  - `POST /api/auth/login` — вход по `email` и `password`, возвращает `{ token, user }`.
- **Поездки**
  - `POST /api/rides` — создать поездку (требуется JWT водителя), тело: `{ from, to, departureTime }`.
  - `GET /api/rides` — получить поездки текущего водителя (требуется JWT водителя).
  - `GET /api/rides/search?from=&to=` — поиск поездок по направлению (для пассажира, можно без авторизации).
- **Заявки**
  - `POST /api/rides/:rideId/requests` — отправить заявку пассажира на поездку.
  - `POST /api/requests/:requestId/accept` — водитель принимает заявку (JWT водителя обязателен), точка добавляется в маршрут.
  - `POST /api/requests/:requestId/reject` — водитель отклоняет заявку (JWT водителя обязателен).

Все сущности (пользователи, поездки, заявки) теперь хранятся в SQLite‑базе (`server/poputchiki.db`) и не теряются при перезапуске сервера.

## Запуск

1. Установите зависимости backend:

   ```bash
   cd server
   npm install
   ```

2. Запустите сервер:

   ```bash
   npm start
   ```

3. Откройте в браузере:

   `http://localhost:3000`

## Как пользоваться

### Роль водителя

1. В блоке **«Аккаунт»** выберите вкладку **«Регистрация»**, роль «Водитель», укажите имя/email/пароль и зарегистрируйтесь (или войдите, если аккаунт уже есть).
2. После входа вкладка **«Я водитель»** станет активной, а поле «Имя водителя» заполнится из профиля.
3. Заполните маршрут: откуда, куда, время выезда и нажмите **«Создать поездку»** — поездка появится в списке ниже.
4. В блоке **«Заявки от пассажиров»** будут отображаться заявки; можно **принять** или **отклонить**.
5. Под поездкой есть ссылка **«Открыть маршрут в Google Maps»** — откроет реальный маршрут с учётом принятых точек пассажиров.

### Роль пассажира

1. При желании зарегистрируйтесь как пассажир (или просто используйте режим без авторизации).
2. Переключитесь на вкладку **«Я пассажир»**.
3. В блоке поиска укажите направление (откуда/куда) и нажмите **«Найти поездки»** — ниже появятся подходящие поездки.
4. Заполните поля с вашим именем и точками посадки/высадки.
5. Нажмите **«Поехать с этим водителем»** у нужной поездки — заявка отправится водителю.
6. В карточках поездок есть ссылка **«Маршрут в Google Maps»**, которая открывает навигацию по основному маршруту водителя.

После того как водитель примет заявку, у него в разделе поездок точка появится в списке маршрута (как дополнительная точка навигации и будет учтена в ссылке Google Maps).
